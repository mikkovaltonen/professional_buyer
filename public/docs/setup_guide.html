Setup Guide

Professional Buyer AI — Initial configuration for Prompt and Knowledge documents

Overview

The chat experience requires a configured System Prompt. Knowledge documents are optional but highly recommended. Follow the steps below to initialize the environment. You can always revisit this page and print to PDF (Ctrl/Cmd + P).

Prerequisites

  • Logged in to the application
  • Backend is configured by the admin

Minimum to chat

  • Required System Prompt configured
  • Optional Knowledge documents uploaded

1) Configure the System Prompt (Required)

  1. Open Admin from the chat header
  2. Click Open Prompt Manager
  3. Create a new version and paste your prompt text. You can use the default from /public/sample_promtp.md as a starting point.
  4. Save the version. The latest version is used automatically.

Tip: Keep prompts in English for consistency across the UI.

2) Upload Knowledge Documents (Optional)

  1. In Admin, open Manage Knowledge Documents
  2. Upload Markdown or text files with procurement policies, onboarding, or guidelines
  3. Return to the chat; the status bar will show the document count

3) Verify in Chat

  • Status bar shows System Prompt: Configured once ready
  • Knowledge Docs count should reflect your uploads
  • Start chatting when the prompt is configured

Troubleshooting

  • System Prompt: Missing — Open Admin, create and save a prompt version
  • Backend misconfigured — Ask an admin to verify backend configuration
  • No Knowledge Docs — Upload via Admin → Internal Knowledge
